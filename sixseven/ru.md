## Описание

Кофейня «У Шамира» реализует программу лояльности «6–7» на основе схемы разделения секрета Шамира: покупаешь 6 эспрессо — получаешь 6 долей полинома степени 5, восстанавливаешь секрет (`coupon_secret = f(0)`), вводишь его как купон и получаешь флаг через QR-код «седьмого напитка».

Проблема: начальный баланс — **40 монет**, эспрессо стоит **10** — хватает только на **4 эспрессо**, а нужно 6.

## Архитектура

- **Flask-приложение** (`app/app.py`, `app/service.py`): регистрация, авторизация, покупка эспрессо, валидация купона, выдача флага.
- **Rust API** (`api/src/main.rs`): эндпоинты для операций над полиномом/долями. Общая SQLite-база с Flask.
- **Криптографическое состояние** (таблица `users`):
  - `module_q` — 256-битное простое число (модуль конечного поля)
  - `poly_coeffs` — JSON-массив из 6 целых коэффициентов `[a_0, a_1, ..., a_5]`
  - `coupon_secret` — свободный член полинома `a_0 = f(0)`

При каждой покупке эспрессо генерируется случайный UUID `order_id`, вычисляется `share = f(uuid_to_int(order_id)) mod module_q` и сохраняется в БД. x-координата каждой доли — UUID, интерпретированный как 128-битное целое.

## Уязвимости

### 1. Скрытый API с эндпоинтом `set_module`

| Эндпоинт              | Метод | Назначение                                     |
|-----------------------|-------|------------------------------------------------|
| `/api/get_module`     | GET   | Прочитать `module_q` пользователя              |
| `/api/set_module`     | POST  | **Перезаписать** `module_q` пользователя       |
| `/api/calc_shares`    | GET   | Пересчитать полином для всех UUID заказов      |
| `/api/combine_shares` | POST  | Интерполяция Лагранжа по переданным долям      |
| `/api/check_coupon`   | POST  | Оракул: совпадает ли купон с секретом?         |

`set_module` позволяет заменить `module_q` на любое значение `q ∈ [2^63, 2^300)`, проходящее тест Ферма для базисов `{2, 3, 5, 7, 11, 13, 17}`. Коэффициенты полинома и `coupon_secret` в БД **не** меняются.

### 2. Пересчёт полинома утекает секрет

`calc_shares` вычисляет значения полинома с **текущим** `module_q`:

```rust
let share_value = evaluate_polynomial(&coeffs, &x_value, &module_q);
```

Если установить `module_q = q`, где `q` делит UUID некоторого заказа `x_i`, то `x_i ≡ 0 (mod q)`, и:

```
f(x_i) mod q = a_0 + a_1·0 + a_2·0² + ... = a_0 mod q = coupon_secret mod q
```

Это позволяет извлечь `coupon_secret mod q` из **одной доли**, полностью обходя пороговую схему.

### 3. Отклонение сжатых кук в Rust API

Rust API проверяет Flask-куки через HMAC-SHA1, но отклоняет куки со сжатым payload (значение начинается с `.`). Flash-сообщения раздувают сессию и включают сжатие. Достаточно зайти на `/dashboard` после логина, чтобы погасить flash-сообщения и получить компактную несжатую куку, которую API принимает.

## Эксплуатация

### Шаг 1 — Найти «удачный» аккаунт

Регистрируем аккаунты и покупаем по 4 эспрессо. Для каждого аккаунта факторизуем 4 UUID заказов (это 128-битные числа). Нужно, чтобы суммарная битовая длина простых множителей `≥ 2^63` по всем четырём UUID превышала 256 — размер секрета.

Типичный наибольший простой делитель 128-битного числа — ~80 бит (функция Дикмана), поэтому обычно хватает ~3 попыток.

### Шаг 2 — Восстановить `coupon_secret` через КТО

Для каждого UUID с подходящим простым множителем `q ≥ 2^63`:

1. `POST /api/set_module` — установить `module_q = q`.
2. `GET /api/calc_shares` — прочитать долю для этого UUID → `coupon_secret mod q`.

Повторяем для всех подходящих простых, затем применяем **Китайскую теорему об остатках (КТО)** для восстановления `coupon_secret` по модулю произведения всех простых. Так как произведение превышает `2^256`, а `coupon_secret < 2^256`, получаем точное значение.

### Шаг 3 — Получить флаг

1. `POST /api/check_coupon` — проверить восстановленный секрет (опционально).
2. `POST /buy-exclusive` с `coupon=<секрет>` — создаёт эксклюзивный заказ.
3. `GET /orders/<exclusive_id>/qr.svg` — декодировать QR → флаг.

```python
# Факторизация UUID, поиск простого делителя >= 2^63 с тестом Ферма
def find_usable_q(uuid_int):
    factors = factorint(uuid_int)
    for p in factors:
        if p >= (1 << 63) and passes_fermat(p):
            return p
    return None

# Для каждого подходящего UUID: ставим q, получаем доли, извлекаем a_0 mod q
for order_id in order_ids:
    x = uuid.UUID(order_id).int
    q = find_usable_q(x)
    if q is None:
        continue
    client.post(f'/api/set_module?user_id={uid}', json={'q': str(q)})
    shares = client.get(f'/api/calc_shares?user_id={uid}').json()['shares']
    for s in shares:
        if s['id'] == order_id:
            residues.append((q, int(s['share'])))

# КТО для восстановления полного секрета
secret = crt([r[0] for r in residues], [r[1] for r in residues])

# Отправляем купон и получаем флаг
client.post('/buy-exclusive', data={'coupon': str(secret)})
```
