## Описание

Сервис — приёмная комиссия вымышленного института. Нужно отправить YAML-аттестат, привязанный к номеру паспорта. Сервер проверяет:

1. Валидный YAML с правильной структурой (13 предметов)
2. Номер паспорта есть в базе
3. Все оценки в `[1, 5]`
4. **Подпись** — полиномиальный хэш от сырых байт YAML совпадает с сохранённым значением
5. Все оценки должны быть ровно `5`, чтобы получить флаг

В оригинальном аттестате есть тройки и четвёрки — простая замена на пятёрки ломает подпись.

## Хэш

```go
const sigP uint64 = 72057594037927931 // 2^56 − 5, простое

func sigOf(b []byte) uint64 {
    var s uint64
    for _, c := range b {
        s = (s*256 + uint64(c)) % sigP
    }
    return s
}
```

Это классический **полиномиальный rolling hash** (вариант отпечатка Рабина): `H = Σ b[i] · 256^(n-1-i) mod P`. Он **не** криптографический — зная алгебраическую структуру, можно подобрать коллизию.

## Уязвимость

YAML игнорирует комментарии (`# ...`). Можно добавить строку-комментарий в конец изменённого аттестата (все пятёрки) — для парсера ничего не изменится, но сырые байты (и значит хэш) меняются. Эти байты под нашим контролем — а значит, можно дотянуть хэш до любого целевого значения.

## Эксплуатация

После добавления `\n# <префикс>` имеем хэш `h`. Нужно найти `k` печатных ASCII-байт `b[0]..b[k-1]`, таких что:

```
h · 256^k + b[0]·256^(k-1) + ... + b[k-1] ≡ target  (mod P)
```

Пусть `R = (target − h · 256^k) mod P`. Нужно число `V = R + j·P`, раскладывающееся в `k` цифр по основанию 256, каждая в `[32, 126]`.

Так как `P = 256^7 − 5`, старшие цифры `V` почти полностью фиксируются значением `R`. Для некоторых `R` фиксированная цифра выходит за `[32, 126]` — уравнение неразрешимо. Решение: перебираем разные префиксы комментария (например, `# %%%`), пока фиксированные цифры не попадут в допустимый диапазон, затем сканируем значения `j`.

С префиксом `%%%` и 10 переменными байтами решение находится быстро. Итоговый YAML:

```yaml
# Подтвержденный аттестат
паспорт: "1337676769"
оценки:
  физика: 5
  химия: 5
  информатика: 5
  английский: 5
  русский: 5
  литература: 5
  физкультура: 5
  обществознание: 5
  алгебра: 5
  история: 5
  география: 5
  биология: 5
  геометрия: 5

# %%% +pX5D~7~}
```

проходит все проверки и возвращает флаг.

## Главные выводы

- Полиномиальный rolling hash — не MAC: зная модуль и имея управляемый хвост сообщения, всегда можно добить сообщение до любого целевого хэша по модулю `P`.
- «Свободное место» формата (комментарии YAML, пробелы JSON, HTTP trailer) даёт атакующему ровно ту степень свободы, которая нужна для управления некриптографическим хэшем без изменения распаршенного значения.
- Ограничение «байты добавки должны быть печатными ASCII» на практике обходится: при `P ≈ 256^k` сильно зажата только старшая цифра неизвестной — перебором префикса легко загнать её в нужный диапазон.
